# Révision Intra : Initiation au Métier de Spécialiste de Solutions de Données (Cours 01-05)

## 1. Contexte Organisationnel (Cours 01)
*   **Organisation** : Ensemble structuré de ressources (Humaines, Matérielles, Financières).
*   **Mission** : Transformer l'information (donnée) en valeur décisionnelle.
*   **Types d'organisations** :
    *   **Privé** : Rentabilité, compétitivité (ex: analyse comportement d'achat).
    *   **Public** : Efficacité des services, bien-être citoyen (ex: Open Data, santé).
    *   **Startups** : Innovation, croissance rapide (ex: architectures agiles).
    *   **Consultation (SSII)** : Expertise pointue, mandats variés.
*   **Rôle Pivot** : Le spécialiste est un "traducteur" entre le monde Métier (Business) et le monde Technique.

## 2. Tâches et Cycles de Vie (Cours 02)
*   **Modèles de gestion de projet** :
    *   **Cascade (Waterfall)** : Linéaire (Analyse > Conception > Dev > Test > Déploiement). Rigide, documentation exhaustive, mais effet "tunnel".
    *   **Modèle en V** : Symétrie entre conception et validation. Tests préparés dès la conception.
    *   **Agile** : Itératif, feedback continu, adaptation aux changements.
*   **Cycle de vie de la donnée** : Collecte, Nettoyage, Stockage, Transformation, Analyse, Visualisation.

## 3. Rôles et Collaboration (Cours 03)
*   **Écosystème Humain** :
    *   **Architecte** : Plan d'ensemble, choix technologiques (Cloud vs Local), sécurité.
    *   **Analyste d'affaires (BA)** : Traduit les besoins business en règles de calcul.
    *   **Développeur App** : Consomme les données via APIs ou tables propres.
    *   **Testeur (QA)** : Filet de sécurité, cherche les erreurs.
    *   **Chef de projet / Scrum Master** : Gardien du temps et des processus.

## 4. Profil du Spécialiste (Cours 04)
*   **Hard Skills (Savoir-faire)** : Modélisation, SQL/NoSQL, ETL (Pipelines), Cloud, Sécurité.
*   **Soft Skills (Savoir-être)** : Communication, Collaboration, Éthique, Rigueur.
*   **Éthique et Légal** : Protection des données (RGPD/Loi 25), confidentialité, intégrité.

## 5. IA et Productivité (Cours 05 & TP2)
*   **IA Générative (GenAI)** : Utilisation des LLMs (GPT, Claude, Gemini) pour automatiser :
    *   Nettoyage de données.
    *   Génération de code (SQL, Python).
    *   Documentation.
*   **Prompt Engineering** : Composantes d'un prompt efficace (Rôle, Contexte, Tâche, Contraintes, Format).
*   **Qualité des Données** : Identification des doublons, valeurs manquantes, formats invalides, incohérences.
