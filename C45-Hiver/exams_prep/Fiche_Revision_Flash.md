# Fiche de Révision Flash (15-30 min avant l'examen)

**Objectif :** Rappeler les réflexes de survie et éviter les erreurs bêtes.

## 1. Check-list Anti-Erreurs (Le mur de la honte)
- [ ] **Les 2 points `:`** : Est-ce qu'ils sont là après mon `if`, `else`, `while`, `for`, `def` ?
- [ ] **L'indentation** : Est-ce que mon bloc de code est bien décalé vers la droite ?
- [ ] **Les parenthèses** : Est-ce que chaque `(` a son `)` ?
- [ ] **Le `=` vs `==`** :
    - `x = 10` (Je mets 10 dans x)
    - `if x == 10:` (Je demande si x est égal à 10)
- [ ] **Les guillemets** : `"Texte"` ou `'Texte'`, mais pas un mélange des deux.

## 2. Rappels Rapides
- **Saisie de nombre** : `int(input(...))` pour entier, `float(input(...))` pour décimal.
- **Boucle simple** : `for i in range(5):` tourne 5 fois (de 0 à 4).
- **Liste** : `ma_liste[0]` est le PREMIER élément.
- **Modulo `%`** : `x % 2 == 0` veut dire que x est PAIR.

## 3. Le réflexe "Détection d'Erreurs"
Si on vous donne un code à corriger, cherchez dans cet ordre :
1. Majuscules au début des mots-clés (`If`, `For`, `While`, `Input`, `Print` sont FAUX).
2. Manque de `:` à la fin d'une ligne de structure.
3. Mauvaise indentation.
4. Manque de conversion (`int` ou `float`) sur un `input`.
5. Mélange de types (ex: `10 + "5"`).

## 4. Schéma Mental (S.I.P.)
1. **S**aisie (input)
2. **I**nstruction (calcul / if / for)
3. **P**rint (affichage final)
